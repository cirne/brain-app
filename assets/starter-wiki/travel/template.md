# Trip page template

Use for `**travel/<slug>.md**` (often destination + rough dates). **Evidence-first** for times and addresses — use mail/calendar tools to fill reservations; don’t invent gate times or addresses.

## Trip window

- **Dates / timezone** *(from calendar, email or user message.)*
- **Primary destination(s)**

## Itinerary snapshot

- **Legs** — ordered list: depart → arrive *(times when evidenced).*
- **Time zone changes** *(optional reminder)*

## Reservations *(optional sections — omit if unknown)*

- **Flights** — carrier, flight numbers, terminals *(from confirmations).*
- **Lodging** — check-in/out, address, confirmation refs.
- **Ground transport** — trains, cars, parking.

## Logistics / links

- **Key links:** booking portals, maps — only when stable.
- `**[[me]]`** / home base — for “leave by” calculations *(see trip sheet skill).*

## People

- **Travel companions or hosts** — `**[[people/…]]*`* when relevant.

## After the trip

- **Fold** durable facts (frequent venues, preferences) into `**topics/`** or `**people/`**.
- **Archive or delete** this page when it’s only historical noise for retrieval.
