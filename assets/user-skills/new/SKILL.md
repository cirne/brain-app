# New wiki page

## When to use

The user wants a **new** markdown page in their wiki, placed correctly and aligned with how the rest of the wiki is organized.

## Process

1. **Discover** — Use search and read tools (wiki, email, web as needed) to see what already exists on the topic or related topics.
2. **Place** — Pick a path under the wiki root that matches existing conventions (folders, naming). Avoid duplicate pages; link out instead.
3. **Write** — Create the page with a clear title, short intro, structured sections, and wikilinks to related pages.
4. **Cross-check** — Mention related people, projects, or emails only when grounded in tool results or the open wiki context.

## Quality bar

- No orphan pages without at least one inbound link from a sensible index or related page (add a link from an existing page when appropriate).
- Prefer one canonical page per topic; merge or link rather than copy-paste.
