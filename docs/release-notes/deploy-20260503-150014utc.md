# Braintunnel release notes

**Release date:** 2026-05-03

**Changes since:** deploy-20260501-162224utc

This release focuses on improving wiki sharing capabilities with a unified namespace, refining the UI with a Tailwind-first approach, and enhancing chat and inbox usability. We also introduce a dev tenant soft reset feature and polish token meter tooltips.

---

## What's new

- **Unified Wiki Sharing Namespace** — Introduced a single `wikis/` root per tenant with `wikis/me/` for owner vaults and `wikis/@<handle>/` for accepted shares, simplifying share projections and management.
- **Wiki Sharing UI & Invite Flow** — Replaced email invite flow with direct accept-by-id API, added WikiShareDialog enhancements, and improved share row UI with vault path labels.
- **Tailwind Migration Progress** — Unified client UI components under a Tailwind-first tree, removed legacy CSS, and migrated many key components (AgentChat, Assistant, BrainHub, onboarding) to Tailwind.
- **Chat and Inbox Improvements** — Added TTS teardown on new chat turns, flattened inbox thread chrome for a cleaner look, and compacted token meter tooltips for clarity.
- **Dev Tenant Soft Reset** — Added a dev-only route to wipe tenant chats, wikis, caches, and shares for QA and testing.

**Also in this release:** minor UI polish including rounded chips and badges, safer wiki share symlink handling, improved calendar tool updates, and documentation updates on wiki sharing architecture and migration.
