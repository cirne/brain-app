# Braintunnel release notes
**Release date:** 2026-05-05
**Changes since:** deploy-20260505-210758utc

This release improves calendar management in the Hub with a more seamless autosave experience and smarter calendar ID handling.

---

## What's new

- **Calendar autosave** — The CalendarPicker now saves changes automatically when toggled, removing the need for a Save button and streamlining your workflow.
- **Primary calendar ID resolution** — Google Calendar's "primary" calendar is now resolved to the actual calendar ID in the Hub UI for more accurate calendar selection.
- **Improved save reliability** — A queue mechanism manages rapid toggle actions during save to prevent conflicts or data loss.
- **UI stability enhancements** — Calendar details remain mounted during reloads after saving, providing a smoother user experience without flicker.

Also in this release: TypeScript typings fixed for Promise resolve in tests and removal of unused calendar field length functions for cleaner code.
